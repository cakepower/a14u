# Brand Guideline

> 타겟 사용자 페르소나, 커뮤니케이션 원칙, 도메인 용어.
> 콘텐츠 작성, 리포트 생성, 전략 제안 시 이 파일을 참조하세요.
> 서비스 개요와 데이터 방법론은 `business-context.md`를 참조하세요.

---

## 1. 타겟 사용자 & 페르소나

### 핵심 페르소나: "이제 막 시작하려는 1인 호스트"

- Airbnb를 막 시작했거나 시작을 준비 중인 호스트
- 가격을 얼마로 설정해야 할지 모르는 상태
- 주변 경쟁 매물이 어떻게 운영되는지 파악하지 못함
- 데이터보다 감에 의존해 운영 중
- 수익을 높이고 싶지만 어디서부터 시작해야 할지 모름

### 주요 고민 (Pain Points)

| 고민 | 상세 내용 |
|------|----------|
| 가격 설정 불확실 | 너무 비싸면 예약이 안 올까봐, 너무 싸면 손해일까봐 — 적정선을 모름 |
| 예약이 안 들어옴 | 등록은 했는데 노출이 되는지, 가격이 문제인지, 사진이 문제인지 파악 못함 |
| 리뷰/평점 관리 모름 | 초반 리뷰를 어떻게 쌓아야 하는지, 평점이 낮으면 어떻게 회복하는지 모름 |

### 서비스 사용 시점

매물 등록 전 사전 조사 단계가 주된 진입점:
- "내 매물 주변 시세가 어느 정도야?"
- "비슷한 매물들은 얼마에 올라와 있어?"
- "지금 시작하면 경쟁이 너무 심한 건 아닐까?"

→ 의사결정 전 불안감을 해소하고, 자신감을 얻기 위해 사용

### 사용자 행동 패턴

- 분석 리포트보다 **결론과 추천**을 먼저 봄
- 숫자보다 **"평균보다 높다 / 낮다"** 같은 상대적 표현을 선호
- 한 번 보고 끝내는 게 아니라 **등록 직전에 다시 확인**하는 경향
- 모바일로 접근할 가능성 높음

### UX 설계 원칙

- 전문 용어(ADR, RevPAR 등)는 반드시 쉬운 설명 병기
- 분석 결과는 "요약 → 인사이트 → 액션" 순서로 제공
- 숫자 나열보다 **시각화(차트, 비교표)** 우선
- 불안감 해소 톤 — 겁주지 않고 "이렇게 하면 돼요" 식으로

## 2. 보이스 & 톤

### 기본 말투 원칙

상황에 따라 톤을 혼용한다.

| 상황 | 톤 | 예시 |
|------|-----|------|
| 분석 결과 / 수치 설명 | 전문적 (`~합니다`) | "현재 동일 행정동 평균 ADR은 78,000원입니다." |
| 전략 제안 / 액션 권고 | 친근 (`~해요`, `~하세요`) | "주말 가격을 10% 올려보세요." |
| 경고 / 주의사항 | 전문적 (`~합니다`) | "해당 데이터는 추정값으로 실제와 다를 수 있습니다." |
| 요약 / 핵심 정리 | 친근 (`~해요`) | "한 줄 요약: 지금 가격은 조금 낮은 편이에요." |

### 전략 제안 구성 방식

**문제 제시 → 원인 → 해결책** 순서를 항상 따른다.

```
[문제]
예약률이 주변 평균(62%)보다 낮은 43% 수준입니다.

[원인]
현재 1박 가격(85,000원)이 동일 조건 경쟁 매물 평균(71,000원)보다
약 20% 높게 설정되어 있어요.

[해결책]
초반 3개월은 가격을 주변 평균 수준(70,000~75,000원)으로 낮춰
예약과 리뷰를 먼저 쌓는 전략을 추천해요.
```

### 글쓰기 규칙

**수치 표현**
- 단독 숫자보다 **비교 표현** 우선: "78,000원" → "주변 평균보다 12% 높은 78,000원"
- 퍼센트와 절댓값 **함께 표기**: "15% 높음 (+11,000원)"
- 추정값은 반드시 표기: "추정 점유율 약 58%"

**강도 표현** — 막연한 형용사 대신 기준이 있는 표현 사용

| 사용 금지 | 대체 표현 |
|----------|----------|
| "가격이 높습니다" | "주변 평균 대비 20% 높습니다" |
| "경쟁이 심합니다" | "동일 행정동 내 활성 매물 47개로 경쟁이 높은 편이에요" |
| "좋은 성과입니다" | "점유율 71%로 주변 평균(58%)을 웃돌고 있어요" |

**액션 표현**
- 제안은 항상 **구체적인 숫자 포함**: "가격을 올리세요" → "65,000원 → 72,000원으로 올려보세요"
- 우선순위 명확히: "먼저 A를 하고, 그 다음 B를 하세요"
- 이유 없는 제안 금지: 반드시 근거 한 줄 이상 포함

**피해야 할 표현**

| 피할 표현 | 이유 |
|----------|------|
| "데이터가 부족하여 분석이 어렵습니다" | 초보 호스트에게 불안감 조성 |
| "시장 상황에 따라 다를 수 있습니다" | 모호한 면피성 표현 |
| "좋은 선택입니다 / 훌륭합니다" | 근거 없는 칭찬 |
| 전문 용어 단독 사용 | ADR, RevPAR 등은 반드시 쉬운 설명 병기 |

### 리포트 구조 템플릿

```
1. 한 줄 요약 (가장 중요한 인사이트 1개)
2. 시장 현황 (수치 + 비교)
3. 내 매물 포지션 (강점 / 약점)
4. 문제 → 원인 → 해결책 (전략 제안)
5. 지금 당장 할 수 있는 액션 (1~3개)
```

## 3. 도메인 용어 사전

| 용어 | 영문 | 정의 | 초보 호스트용 설명 |
|------|------|------|-------------------|
| ADR | Average Daily Rate | 예약된 날의 평균 1박 가격 | "실제로 팔린 날의 평균 숙박비" |
| 점유율 | Occupancy Rate | 전체 가능 일수 중 예약된 날의 비율 (%) | "한 달 중 며칠이 예약됐는지" |
| RevPAR | Revenue Per Available Room | ADR × 점유율 = 실질 수익성 지표 | "가능한 날 기준으로 평균 얼마 벌었는지" |
| 리드타임 | Booking Lead Time | 예약 생성일부터 체크인까지의 기간 | "얼마나 미리 예약이 들어오는지" |
| 최소 박수 | Minimum Stay | 예약 가능한 최소 숙박 일수 | "1박 예약 받을지, 2박 이상만 받을지" |
| 블로킹 | Blocking | 호스트가 직접 날짜를 예약 불가로 막는 것 | "내가 쓸 날짜 막아두기" |
| 즉시예약 | Instant Book | 호스트 승인 없이 게스트가 바로 예약하는 방식 | "자동 수락 설정" |

### 포지셔닝 용어

| 용어 | 정의 |
|------|------|
| 가성비 포지션 | 주변 대비 낮은 가격 + 충분한 품질로 점유율 극대화 |
| 프리미엄 포지션 | 차별화된 시설/서비스로 높은 ADR 유지 |
| 틈새 포지션 | 특정 게스트(반려동물, 장기투숙 등) 타겟 특화 |
| 밸런스 포지션 | ADR과 점유율의 균형 — RevPAR 최적화 |

### 자주 혼동되는 개념

- **가격(정가) ≠ ADR**: 정가는 내가 설정한 가격, ADR은 실제로 팔린 가격의 평균
- **점유율 높음 ≠ 수익 높음**: 너무 낮은 가격으로 채우면 RevPAR가 낮을 수 있음
- **리뷰 수 ≠ 평점**: 리뷰가 많아도 평점이 낮으면 노출에 불리
